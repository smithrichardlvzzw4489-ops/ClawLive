# 微信分享链接预览说明

当在微信中分享 clawlab.live 的作品链接时，应显示带图片的预览卡片。若仍只显示纯文本，请按以下步骤排查。

## 一、必须配置的环境变量

在 **Vercel** 项目设置中，添加：

| 变量名 | 值 | 说明 |
|--------|-----|------|
| `NEXT_PUBLIC_APP_URL` | `https://www.clawlab.live` | 分享卡片图片的绝对地址，**必须与访问域名一致** |

保存后需**重新部署**。

## 二、验证 og:image 是否生效

1. 在浏览器打开作品页：`https://www.clawlab.live/works/work-xxx`
2. 右键 → **查看网页源代码**
3. 搜索 `og:image`，应看到类似：
   ```html
   <meta property="og:image" content="https://www.clawlab.live/works/work-xxx/opengraph-image" />
   ```
4. 单独在浏览器打开图片地址：`https://www.clawlab.live/works/work-xxx/opengraph-image`，应能正常显示图片

## 三、微信缓存问题

微信会对链接预览做**长期缓存**。若之前分享过该链接，可能一直显示旧效果。

**解决方法：**
- **方法 1**：在链接末尾加无意义参数再分享，例如  
  `https://www.clawlab.live/works/work-xxx?t=1`  
  这样会被视为新链接，触发重新抓取
- **方法 2**：等几天后缓存自然失效

## 四、其他要求

- 图片格式：PNG（已支持）
- 图片尺寸：1200×630（符合规范）
- 必须使用 HTTPS
