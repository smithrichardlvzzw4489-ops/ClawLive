# 微信分享链接预览 - 完整排查指南

## 一、当前验证结果（已确认正常）

已对 `https://www.clawlab.live/works/work-xxx` 进行验证：

| 检查项 | 结果 | 说明 |
|--------|------|------|
| og:image meta 标签 | ✅ 存在 | HTML 源码中包含完整绝对 URL |
| og:title / og:description | ✅ 存在 | 标题和描述正确 |
| 图片 URL 可访问性 | ✅ 200 OK | 直接访问返回 image/png，约 220KB |
| 模拟微信爬虫请求 | ✅ 200 OK | User-Agent: MicroMessenger 时仍返回 200 |
| 模拟 Facebook 爬虫 | ✅ 200 OK | 页面请求正常 |

**结论**：从境外网络测试，元数据和图片均正常。若微信仍无预览，多半与网络环境或缓存有关。

## 二、最可能原因：中国大陆网络

**clawlab.live 部署在 Vercel**，主要面向海外。微信的抓取服务器在中国大陆：

- Vercel 在国内访问可能较慢或被限制
- 微信爬虫请求 `clawlab.live` 时，可能超时或失败
- 表现：境外分享正常，国内微信仍无预览

**可行方案**：
1. **国内 CDN 回源**：用 Cloudflare、又拍云、阿里云 CDN 等，将域名解析到国内节点并回源到 Vercel
2. **国内部署**：把前端部署到阿里云、腾讯云等国内服务器
3. **接受现状**：若主要用户在国外，可暂不处理

## 三、微信缓存

微信会对链接预览做**长期缓存**（有时达数周）。

**处理方式**：
- 使用带参数的新链接分享，例如：`https://www.clawlab.live/works/work-xxx?v=2`
- 等待几天后再试同一链接

## 四、必须配置的环境变量

在 **Vercel** 中设置：

| 变量名 | 值 | 说明 |
|--------|-----|------|
| `NEXT_PUBLIC_APP_URL` | `https://www.clawlab.live` | 生成 og:image 的绝对地址 |
| `NEXT_PUBLIC_API_URL` | 你的 Railway 后端地址 | 动态生成 OG 图时拉取作品信息 |

保存后需**重新部署**。

## 五、自检步骤

1. **查看网页源码**
   - 打开作品页 → 右键 → 查看网页源代码
   - 搜索 `og:image`，确认类似：
     ```html
     <meta property="og:image" content="https://www.clawlab.live/works/work-xxx/opengraph-image..." />
     ```

2. **直接访问图片 URL**
   - 在浏览器打开：`https://www.clawlab.live/works/work-xxx/opengraph-image`
   - 应能看到 1200×630 的预览图

3. **使用 Facebook 分享调试**
   - 打开：https://developers.facebook.com/tools/debug/
   - 输入作品链接，查看抓取结果
   - 若 Facebook 正常，微信理论上也应支持

## 六、技术说明

- **动态图**：`/works/[workId]/opengraph-image`，调用 API 拉取作品信息生成
- **静态兜底图**：`/og-default`，无 API 依赖，仅作备用
- **图片规格**：1200×630 PNG，符合常见分享要求
