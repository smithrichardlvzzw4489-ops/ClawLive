# ClawLive 后端部署指南

前端已部署在 Vercel，后端需单独部署。后端是 **Express + Socket.io** 长连接服务，不能部署在 Vercel（Vercel 为无状态 serverless）。

---

## 一、部署前准备

### 1. 后端依赖

| 服务 | 是否必需 | 说明 |
|------|----------|------|
| **PostgreSQL** | ✅ 必需 | 用户、房间、消息等数据 |
| **Redis** | ⚠️ 可选 | Socket.io 多实例扩展用，单实例可不用 |
| **Node.js 20+** | ✅ 必需 | 运行环境 |

### 2. 环境变量清单

| 变量名 | 必需 | 说明 |
|--------|------|------|
| `DATABASE_URL` | ✅ | PostgreSQL 连接串，如 `postgresql://user:pass@host:5432/dbname` |
| `JWT_SECRET` | ✅ | JWT 密钥，建议 32 位以上随机字符串 |
| `JWT_REFRESH_SECRET` | ✅ | 刷新令牌密钥 |
| `CORS_ORIGIN` | ✅ 生产 | 前端域名，如 `https://claw-live-web-x82m.vercel.app`（多域名用逗号分隔） |
| `REDIS_URL` | ⚠️ | Redis 连接串，单实例可不配 |
| `WEBHOOK_SECRET` | ⚠️ | Webhook 验证密钥 |
| `SERVER_PORT` | ⚠️ | 端口，默认 3001，云平台一般自动注入 `PORT` |

## 二、推荐方案：Railway 部署（最简单）

Railway 支持一键 PostgreSQL、一键部署，适合快速上线。

### 步骤 1：注册 Railway

1. 打开 https://railway.app/
2. 用 GitHub 登录

### 步骤 2：创建项目并添加 PostgreSQL

1. 点击 **New Project**
2. 选择 **Provision PostgreSQL**，自动创建数据库
3. 进入 PostgreSQL 服务，点击 **Variables**，复制 `DATABASE_URL`

### 步骤 3：添加 Redis（可选）

1. 在项目中点击 **+ New**
2. 选择 **Database** → **Redis**
3. 复制 `REDIS_URL`（格式类似 `redis://default:xxx@xxx.railway.internal:6379`）

> 若不用 Redis，后端会以单机模式运行 Socket.io，功能正常，只是多实例时无法共享会话。

### 步骤 4：部署后端代码

1. 在项目中点击 **+ New** → **GitHub Repo**
2. 选择你的 ClawLive 仓库
3. 在服务设置中：
   - **Root Directory**：留空（从仓库根目录构建）
   - **Build Command**：`pnpm install && pnpm exec turbo run build --filter=@clawlive/server`
   - **Start Command**：`cd apps/server && npx prisma migrate deploy && node dist/index.js`
   - **Watch Paths**：`apps/server/**`（可选，用于增量部署）

4. 在 **Variables** 中添加：

   ```
   DATABASE_URL=<从 PostgreSQL 服务复制的连接串>
   JWT_SECRET=<随机 32 位字符串>
   JWT_REFRESH_SECRET=<另一个随机字符串>
   CORS_ORIGIN=https://claw-live-web-x82m.vercel.app
   NODE_ENV=production
   ```

   > 若用了 Redis，再加上 `REDIS_URL`。

5. Railway 会注入 `PORT`，需在代码中支持。检查 `apps/server/src/index.ts` 是否使用 `process.env.PORT || process.env.SERVER_PORT || 3001`。
   - 若未支持，可添加变量：`SERVER_PORT` = Railway 分配给你的端口（如 3001），或修改代码读取 `PORT`。

### 步骤 5：检查端口配置

Railway 会设置 `PORT` 环境变量。请确保后端使用它：
- 打开 `apps/server/src/index.ts`
- 将 `const PORT = process.env.SERVER_PORT || 3001;` 改为：`const PORT = process.env.PORT || process.env.SERVER_PORT || 3001;`

### 步骤 6：获取公网地址

1. 在服务设置中点击 **Settings** → **Networking** → **Generate Domain**
2. 记下域名，例如 `clawlive-server-production.up.railway.app`

### 步骤 7：配置 Vercel 前端

在 Vercel 项目 **Settings** → **Environment Variables** 中增加：

```
NEXT_PUBLIC_API_URL=https://clawlive-server-production.up.railway.app
NEXT_PUBLIC_SOCKET_URL=https://clawlive-server-production.up.railway.app
```

保存后重新部署前端。

## 三、备选方案：Render 部署

1. 打开 https://render.com/
2. 新建 **Web Service**，连接 GitHub 仓库
3. 配置：
   - **Root Directory**：留空
   - **Build Command**：`pnpm install && pnpm exec turbo run build --filter=@clawlive/server`
   - **Start Command**：`cd apps/server && npx prisma migrate deploy && node dist/index.js`
4. 添加 **PostgreSQL** 数据库（同一项目内），在 Web Service 中引用其 `DATABASE_URL`
5. 配置环境变量（同上），包括 `CORS_ORIGIN`

Render 会注入 `PORT`，同样需在代码中支持 `process.env.PORT`。

---

## 四、备选方案：Docker 自托管

若你有 VPS（如阿里云、腾讯云），可用 Docker 部署：

```bash
# 克隆仓库
git clone <your-repo-url>
cd ClawLive

# 创建 .env（参考 .env.example 和 .env.cloud-template）
# 必须设置：DATABASE_URL, JWT_SECRET, CORS_ORIGIN（包含你的前端域名）

# 仅启动后端相关服务
docker-compose up -d postgres redis server

# 查看日志
docker-compose logs -f server
```

前端 `.env` 中设置 `NEXT_PUBLIC_API_URL` 和 `NEXT_PUBLIC_SOCKET_URL` 为你的服务器地址。

## 五、本地验证部署

部署完成后，可用以下方式验证：

1. **健康检查**：访问 `https://你的后端域名/health`，应返回 `{"status":"ok","timestamp":"..."}`
2. **前端登录**：在前端输入用户名密码，若能成功登录，说明 API 和 CORS 正常

## 六、常见问题

### 1. 前端仍提示「无法连接服务器」

- 检查 Vercel 环境变量 `NEXT_PUBLIC_API_URL` 是否指向后端地址
- 重新部署前端（环境变量修改后需重新 build）
- 确认后端 CORS 中包含前端域名

### 2. CORS 错误

- 确保 `CORS_ORIGIN` 包含前端完整 URL，如 `https://claw-live-web-x82m.vercel.app`
- Vercel 可能有多域名（如 `xxx.vercel.app` 和自定义域名），都需加入 `CORS_ORIGIN`，用逗号分隔

### 3. Prisma 迁移失败

- 确认 `DATABASE_URL` 正确且有写权限
- 本地执行：`cd apps/server && pnpm prisma migrate deploy` 查看具体错误

### 4. Socket.io 连接失败

- 确认 `NEXT_PUBLIC_SOCKET_URL` 与 `NEXT_PUBLIC_API_URL` 一致（同一后端）
- 检查浏览器控制台是否有 WebSocket 相关错误

---

## 七、安全检查清单

- [ ] `JWT_SECRET` 和 `JWT_REFRESH_SECRET` 为随机强密码，非示例值
- [ ] `CORS_ORIGIN` 只包含信任的前端域名
- [ ] `DATABASE_URL` 不要泄露到前端或日志
- [ ] 生产环境 `NODE_ENV=production`
